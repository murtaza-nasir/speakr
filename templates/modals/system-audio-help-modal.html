macOS doesn't let any browser capture system audio directly — the OS treats audio capture as a privileged operation that browsers (sandboxed apps) can't request. The standard workaround is a free virtual audio driver that routes the system output back into an input device the browser can read like a microphone.

Option A — Tab audio (no install)
  1. Open the source content in a tab (YouTube, Zoom web, etc.)
  2. Click System Audio in Speakr and pick the tab in the share dialog
  3. Tick Share tab audio before clicking Share

Works only for content playing in a browser tab. App audio (Slack, native Zoom client, etc.) won't be captured.

Option B — Virtual audio device (recommended)
  1. Install BlackHole 2ch (free) — installer signs in, no kernel-extension prompts.
  2. Open Audio MIDI Setup → create a Multi-Output Device containing your speakers + BlackHole. Set this as the system output.
  3. In Speakr, click Microphone and pick BlackHole 2ch as the input.

After this is set up, system audio captures continuously regardless of which tab/window is active.

Alternative paid tools: Loopback (Rogue Amoeba) is the most polished UX.

Windows is the easiest platform — Chrome and Edge can capture audio from any source the OS plays, including specific apps. No extra software needed.

System-wide capture
  1. Click System Audio or Mic + System in Speakr
  2. In the share dialog, pick Entire Screen
  3. Tick Share system audio (bottom-left of the dialog)
  4. Click Share
Per-window capture
  1. Pick Window instead of Entire Screen
  2. Choose the specific app's window
  3. Tick Share window audio

Captures only the chosen app's audio — useful for meeting recordings without your own keyboard/typing noise.

Stuck on "no audio track" errors? Some apps (Spotify desktop, DRM-protected video) suppress system-audio capture. Use a virtual cable like VB-Audio Cable as a workaround.

Linux has two distinct paths. Pick the one that matches what you actually want to capture:

Option A — Tab audio (no setup)
  1. Play the source content in a Chrome tab (YouTube, Zoom web, meeting URL, etc.)
  2. Click System Audio in Speakr — pick that tab in the share dialog
  3. Tick Share tab audio before clicking Share
This won't show up in pavucontrol. Chrome captures tab audio internally and never routes through PulseAudio, so the Recording tab will say "No application is currently recording audio." That's normal — Speakr's meter is the source of truth.

Only captures audio playing inside a browser tab. Native apps (Slack, Spotify desktop, Discord) won't be picked up.

Option B — Full system audio via PulseAudio monitor (one-time setup)

Every Linux audio sink has a corresponding "monitor" source you can record FROM. Chrome can pick it as a microphone input — then anything playing through your speakers gets captured. Use the Microphone button, not System Audio.

  1. Open pavucontrol
  2. In Configuration tab, make sure your output device profile is set (HDMI / Analog Stereo / etc.)
  3. In Speakr, click Microphone and grant permission. This makes Chrome enumerate input devices including monitor sources.
  4. While Speakr is recording, open pavucontrol's Recording tab. Find Speakr's capture entry and switch its source from your microphone to Monitor of <your output>
  5. Subsequent records will remember this routing if you make it the default.

To capture BOTH your mic and the monitor, create a virtual combined source: pactl load-module module-null-sink sink_name=combined then loopback both inputs to it.

PipeWire (newer distros)

Same monitor-source pattern. Use qpwgraph or helvum to wire your playback sink's monitor into Chrome's capture input.
